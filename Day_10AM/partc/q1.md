# Time complexity
- The time complexity for lookup,update and insert is of O(1) on avg and O(n) on worst cases
- 
# avg tme complexity - o(1)

A dictionary uses a hash table:
- Python computes a hash value of the key.
- The hash determines the index in an internal array.
- The value is stored or retrieved directly from that index.

# o(n) tme complexity

- Worst case happens when many keys produce the same hash index. This is called a hash collision.

# hashing for integers and strings

- For integers, hashing is done by hash(x) = x
- Python uses a polynomial rolling hash.Each string produces a large unique integer
