## Step 1: Exact Prompt Used

> *"Write a Python function that takes two dictionaries representing student grades from two different semesters and produces a merged report showing: combined GPA, grade trend (improving/declining/stable), and subjects common to both semesters. Use defaultdict and dict comprehension."*

---

## Step 2: AI's Output Code (Raw)

```python
from collections import defaultdict

def merge_student_grades(sem1: dict, sem2: dict) -> dict:
    all_subjects = defaultdict(list)

    for subject, grade in sem1.items():
        all_subjects[subject].append(grade)
    for subject, grade in sem2.items():
        all_subjects[subject].append(grade)

    combined_gpa = sum(
        sum(grades) / len(grades) for grades in all_subjects.values()
    ) / len(all_subjects)

    sem1_avg = sum(sem1.values()) / len(sem1)
    sem2_avg = sum(sem2.values()) / len(sem2)

    if sem2_avg > sem1_avg:
        trend = "improving"
    elif sem2_avg < sem1_avg:
        trend = "declining"
    else:
        trend = "stable"

    common_subjects = {
        subject: {"sem1": sem1[subject], "sem2": sem2[subject]}
        for subject in sem1 if subject in sem2
    }

    return {
        "combined_gpa": round(combined_gpa, 2),
        "trend": trend,
        "common_subjects": common_subjects
    }
```

## Step 3: Critical Evaluation

| Check                      | Result | Finding                                                                                                                                                        |
|----------------------------|:------:|----------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Handles missing subjects?  |   ❌    | `combined_gpa` divides by `len(all_subjects)` — this averages *subject averages*, not individual grades, skewing GPA when subjects appear in only one semester |
| Uses `.get()` safely?      |   ❌    | `sem1.values()` and `sem2.values()` are accessed directly — causes `ZeroDivisionError` on empty dicts                                                          |
| Trend calculation correct? |   ⚠️   | Concept is right, but exact floating-point equality (`==`) means values like `85.0000001` vs `85.0` never register as `"stable"`                               |
| Handles edge cases?        |   ❌    | Empty dicts crash with `ZeroDivisionError`. Single-semester input (one dict empty) also crashes. No input validation at all                                    |
| Pythonic?                  |   ✅    | The `common_subjects` dict comprehension is clean and the `defaultdict` is used appropriately. GPA loop could be a cleaner list comprehension                  |

### Summary of Problems

1. **Wrong GPA formula** — averaging per-subject averages ≠ averaging all grades
2. **No empty dict guard** — division by zero crashes the function
3. **Floating-point "stable" never triggers** — no tolerance/epsilon used
4. **No `.get()` safety** — direct key access inside dict comprehension risks `KeyError`
5. **No type hints or docstrings** — not production-ready

## Step 4: Improved Version

```python
"""
grade_merger.py
---------------
Improved solution for Part D — AI-Augmented Task.
Merges two semester grade dictionaries into a unified student report.
"""

from collections import defaultdict


# ── Type alias for clarity ──────────────────────────────────────────────────
GradeDict = dict[str, float]


def merge_student_grades(sem1: GradeDict, sem2: GradeDict) -> dict:
    """
    Merge two semester grade dictionaries into a consolidated student report.

    Args:
        sem1: Mapping of subject -> numeric grade for Semester 1.
              Example: {"Math": 88.0, "Physics": 74.0}
        sem2: Mapping of subject -> numeric grade for Semester 2.
              Example: {"Math": 91.0, "Chemistry": 82.0}

    Returns:
        A dict containing:
            - "combined_gpa"    (float | None): Mean of all grades across both
                                semesters. None if both dicts are empty.
            - "trend"           (str): "improving", "declining", "stable", or
                                "insufficient data" when a semester is empty.
            - "common_subjects" (dict): Subjects present in both semesters,
                                mapped to {"sem1": grade, "sem2": grade,
                                           "change": delta}.
            - "sem1_avg"        (float | None): Semester 1 average grade.
            - "sem2_avg"        (float | None): Semester 2 average grade.

    Raises:
        TypeError:  If either argument is not a dict.
        ValueError: If any grade value is not a real number.

    Examples:
        >>> s1 = {"Math": 80.0, "English": 70.0}
        >>> s2 = {"Math": 90.0, "English": 85.0}
        >>> r  = merge_student_grades(s1, s2)
        >>> r["trend"]
        'improving'
        >>> r["common_subjects"]["Math"]["change"]
        10.0
    """
    # ── Input validation ────────────────────────────────────────────────────
    if not isinstance(sem1, dict) or not isinstance(sem2, dict):
        raise TypeError("Both arguments must be dictionaries.")

    for label, grades in (("sem1", sem1), ("sem2", sem2)):
        for subject, grade in grades.items():
            if not isinstance(grade, (int, float)):
                raise ValueError(
                    f"{label}[{subject!r}] = {grade!r} is not a numeric grade."
                )

    # ── Aggregate all grades per subject using defaultdict ──────────────────
    all_grades: defaultdict[str, list[float]] = defaultdict(list)

    for subject, grade in sem1.items():
        all_grades[subject].append(float(grade))
    for subject, grade in sem2.items():
        all_grades[subject].append(float(grade))

    # ── Combined GPA: mean of every individual grade (not subject averages) ─
    flat_grades: list[float] = [g for grades in all_grades.values() for g in grades]
    combined_gpa: float | None = (
        round(sum(flat_grades) / len(flat_grades), 2) if flat_grades else None
    )

    # ── Per-semester averages (guard against empty dicts) ───────────────────
    sem1_avg: float | None = (
        round(sum(sem1.values()) / len(sem1), 2) if sem1 else None
    )
    sem2_avg: float | None = (
        round(sum(sem2.values()) / len(sem2), 2) if sem2 else None
    )

    # ── Trend: use a small epsilon to treat tiny FP differences as "stable" ─
    STABLE_THRESHOLD = 0.5   # grade-point tolerance

    if sem1_avg is None or sem2_avg is None:
        trend = "insufficient data"
    elif sem2_avg - sem1_avg > STABLE_THRESHOLD:
        trend = "improving"
    elif sem1_avg - sem2_avg > STABLE_THRESHOLD:
        trend = "declining"
    else:
        trend = "stable"

    # ── Common subjects: dict comprehension, using .get() safely ────────────
    common_subjects: dict = {
        subject: {
            "sem1":   sem1.get(subject),          # .get() — never KeyErrors
            "sem2":   sem2.get(subject),
            "change": round(
                float(sem2.get(subject, 0)) - float(sem1.get(subject, 0)), 2
            ),
        }
        for subject in sem1.keys() & sem2.keys()  # set intersection — Pythonic
    }

    return {
        "combined_gpa":    combined_gpa,
        "trend":           trend,
        "common_subjects": common_subjects,
        "sem1_avg":        sem1_avg,
        "sem2_avg":        sem2_avg,
    }
```

## Summary of All 5 Fixes Made

| Bug in AI Output | Fix Applied |
|---|---|
| `ZeroDivisionError` on empty dicts | Guard all divisions with `if sem1` / `if sem2`; return `None` gracefully |
| Wrong GPA formula (averaged subject-averages) | Flatten all grades into one list before dividing |
| Floating-point `"stable"` never triggers | Added `STABLE_THRESHOLD = 0.5` epsilon for near-equal averages |
| No `.get()` safety in dict comprehension | Used `sem1.get(subject)` so no `KeyError` is ever possible |
| No type hints or docstrings | Added full PEP 484 type hints, `GradeDict` alias, and detailed docstring |

### Bonus Pythonic Improvement

Used `sem1.keys() & sem2.keys()` (set intersection on dict views) instead of
`if subject in sem2` — cleaner and more idiomatic Python.
